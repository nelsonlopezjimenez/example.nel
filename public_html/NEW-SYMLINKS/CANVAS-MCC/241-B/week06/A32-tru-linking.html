Creating a Hypertext Document, Anchors and Links

  1. So what makes the World Wide Web so different from other information media? It's the links!
    • Many experts think that one of the main reasons the World Wide Web has been so successful is because it more closely reflects how our brains work.
    • If you are like most individuals you are not just sitting there reading these notes with nothing else on your brain, instead you have other fleeting thoughts coming in and out. Myself, for example, have had stray thoughts about making sure I pick up my daughter for an appointment this afternoon, checking to see if a hair appointment I made was today or tomorrow... all of those thoughts.
    • So links on Web pages reflect the ability to "see more about it" or allow us to move about the Web in a non-linear mode. A bit like our brains work.
  2. The W3C contains a vast amount of information about hyperlinks and creating them. The standards can be found at: Anchor tag
  3. All though links use http (Hyper Text Transfer Protocol) to call a Web page, you should be aware that links can use too many different Internet protocols and are not restricted to just http. A list of other protocols that you can use is at the end of this document.
  4. Some basic rules about Links and the Anchor element:
    • The Anchor element should be contained within the BODY element of your document
    • The Anchor element is used for creating hypertext links to other documents on the Web or other parts of the current document.
    • The Anchor is a container tag and must include an on and off statement.
      Syntax:

<a href=" " name=" " [target="" rel=" " rev=" "]>description of the link</a>

  • The href attribute
    1. specifies the location of a Web document or resource, such as an ftp file. It defines a link between the current document or to particular paragraphs in another document and the destination document or resources.
    2. is included within the on tag of the ANCHOR element
    3. Acceptable values you can assign to the HREF attribute include any valid URL on the Internet.
      Example:

<A HREF="http://www.cis.edcc.edu/">CIS Homepage</A>

  • The name attribute
    1. identifies the location within the current Web document.
    2. is included within the on tag of the ANCHOR element
    3. Example, you want to allow the individual looking at your page the ability to return to the top of the page with a hyperlink.
    4. For instance, at the top of this dialog there are links to the subject matters within this page.  Hyperlinks using the href attribute create the link.  Hyperlinks using the name attribute create the destination.
    5. To create such a link you need to create two ANCHOR elements.
    6. The name attribute defines your destination, the ending point.
    7. An ANCHOR tag with an HREF attribute defines the hypertext link for your viewer to click on.
      Example:

Creating the destination location:

<a name="top"></A>

Creating the hyperlink to the go there:

<a href="#top">Return to the top</a>

  1. When using the href attribute to access sections of your page, you need to include the "#" before the section name.
  2. Section names are case sensitive, "Top" is not the same as "top"
  3. One of the most common usage's of the section link is return the reader to the top of a page.
  4. You can use the href and the name attributes within the same ANCHOR tag.
  • The target attribute
    1. is used with framed Web sites
    2. is included within the on tag of the ANCHOR element
    3. it specifies the name of a frame where a document is to be opened
    4. we will cover this when we do frames, Chapter 5
  • The rel attribute:
    1. This attribute describes the relationship from the current document to the document specified by the href attribute.
    2. is included within the on tag of the ANCHOR element
    3. This attribute is done for those individuals that look at source code. There are a lot of us out there!
    4. Example: You want the individual viewing your source code to know that the hyperlink takes you to the top level of your Web site.
      Syntax:

<a href="http://www.cis.edcc.edu/" rel="Parent Site">CIS Homepage</a>

  • The rev attribute
    1. is the opposite of the rel attribute.
    2. is included within the on tag of the ANCHOR element
    3. It describes what the current page is to the destination page assigned in the href attribute.
      Syntax:

<a href="http://www.cis.edcc.edu/" rel="Parent Site" rev="Course Site">CIS Homepage</a>

Applying ANCHOR tags

Links can direct an individual to documents within your own site, or documents at other sites. So far, each hyperlink example you have seen is has included the complete address or URL of the document you are pointing at.

It doesn't have to be! You can create links to pages within your own site by using relative links

  1. Linking Near - Creating links to documents within your own site!
    • To understand linking within your own site, let's review the addressing scheme of an individual URL.
    • The complete address for a listing of the magazines, newspapers, and journal articles available at EdCC is: http://www.edcc.edu/library/Databases/default.htm#articles
      1. http: is the protocol we are using to access the page
      2. www.edcc.edu is the Web server we are access, which holds the page. This portion of the address includes the host.second_level_domain.first_level_domain.domain
      3. library/ is the first level/folder/directory of the path on the server where these pages are kept.
      4. Databases/, the second level/folder/directory of the path on the server where these pages are kept.
      5. default.htm is the name of the Web document you are looking for
      6. #articles is the location within the document you want to see.
    • This type of path structure is called the structure of the site!
      1. To create a link to another page within the site, you need to know this structure.
    • To work with this more, let's create a fictitious site.

  • you want to create a link from the first page - http://www.myCompany.com/default.htm to a second page - http://www.myCompany.com/orders/default.htm.
    1. you could type out the full address to create your link: <a href='http://www.myCompany.com/orders/default.htm'>Order Our Products</a>
  • or you can create a relative link to the new file
    1. You need to find the common directory! and use the a shortcut ../
    2. Example: move from http://www.mycompany.com/default.htm to http://www.myCompany.com/orders/default.htm
    3. the common directory in this case is the root directory of myCompany
    4. In this example you want to move down to the directory in which the file is held
    5. list the directory structure down to your file orders/default.htm
  • your completed relative hyperlink looks like:

    <a href="orders/default.htm">Order Our Products</a>

    Note: that, you lose the http://
  • You've created your relative path!
  • Let's try another, move from orders/shipping/shippinginformation.htm to products/helmets/adults.htm
    1. the common directory in this example is again the root directory
    2. you want to move up to that directory so build the first level of your relative link with ../../ one for each level in the structure you want to go up
    3. BUT, the intermediate folders/directories are not the same
    4. now list the directory structure back down to your file Products/helmets/
    5. and finally the file, adults.htm
  • your completed relative hyperlink looks like:

    <a href="../../Products/helmets/adults.htm">Adult Helmets</a>

  • One last example: move from http://www.mycompany.com/products/default.htm to http://www.mycompany.com/information/history.htm
    1. the common directory in this example is root directory
    2. ../, one for each level in the structure you want to go up
    3. BUT, the intermediate folders/directories are not the same
    4. now list the directory structure back down to your file information/history.htm
  • your completed relative hyperlink looks like:

    <a href="../information/history.htm">Company History</a>

  • Many web page editors will create relative hyperlinks for you.
  • Linking Far - Creating hyperlinks to other sites
    • When creating a hyperlink to another site, remember to use the full URL and Internet protocol.
    • Example: <A HREF="http://www.w3c.org/">The W3C</A>
    • you must have the http:// to make a link to another location successfully

Linking to Documents on the Internet

  1. any Internet protocol can be used in replacement of http:// within your links
  2. other Internet protocols:
  PROTOCOL Internet service the protocol access:
https:// Secure HTML documents
file:// HTML documents and file extensions designated by plug-in or helper applications located on your hard or floppy drives.
ftp:// File Transfer Protocol sites and files
gopher:// Gopher menus and documents
news: UseNet newsgroups
mailto: email messages, if your browser has had the mail server identified
telnet: remote log-in access to another computer.
