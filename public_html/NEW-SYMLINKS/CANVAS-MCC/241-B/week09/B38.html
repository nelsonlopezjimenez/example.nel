Definition and Usage

From W3C site.

The <table> tag defines an HTML table.

An HTML table consists of the <table> element and one or more <tr>, <th>, and <td> elements.

The <tr> element defines a table row, the <th> element defines a table header, and the <td> element defines a table cell.

A more complex HTML table may also include <caption>, <col>, <colgroup>, <thead>, <tfoot>, and <tbody> elements.

Note: Tables should not be used for page layout! Historically, some Web authors have misused tables in HTML as a way to control their page layout. However, there are a variety of alternatives to using HTML tables for layout, primarily using CSS.

HTML Table Tags

Tag Description
<table> Defines a table
<th> Defines a header cell in a table
<tr> Defines a row in a table
<td> Defines a cell in a table
<caption> Defines a table caption
<colgroup> Specifies a group of one or more columns in a table for formatting
<col> Specifies column properties for each column within a <colgroup> element
<thead> Groups the header content in a table
<tbody> Groups the body content in a table
<tfoot> Groups the footer content in a table

  1. Differences Between HTML 4.01 and HTML5

    The "align", "bgcolor", "border", "cellpadding", "cellspacing", "frame", "rules", "summary", and "width" attributes are not supported in HTML5. However, you may find old pages, so here are some definitions:

  • The cellspacing attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    1. this attribute controls the spacing between cells of the table
    2. acceptable values to this attribute are positive integers representing pixels
  • The cellpadding attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    1. this attribute controls the spacing between with the cells of the table, the margin between the walls of the cell and the text of the cell
    2. acceptable values to this attribute are positive integers representing pixels
  • The border attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    1. this attribute controls the width of the border of all walls of the table
    2. acceptable values to this attribute are positive integers representing pixels
  • A graphical representation of CELLPADDING, CELLSPACING and BORDER
 

  • The width attribute HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    1. This attribute specifies the desired width of the entire table.
    2. It is intended for visual browsers.
    3. The acceptable values to assign to this attribute can either be pixels or a percentage value.
      • When the value is a percentage value, the value is relative available space of the browser window.
      • When WIDTH is not specified by the author, the browser determines how wide the table will be

    • This attribute specifies the alignment of data and the justification of text within the complete row.
    • Acceptable values to this attribute are: ("left" | "center" | "right" | "justify" | "char")
Value

  3. The align attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    • This attribute specifies the alignment of data and the justification of text the cell.
    • Acceptable values to this attribute are:
      LEFT|CENTER|RIGHT|JUSTIFY|CHAR
  4. The valign attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    • This attribute specifies the vertical position of data within a cell.
    • The acceptable values of this attribute are
      TOP|MIDDLE|BOTTOM|BASELINE
  5. The colspan attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    • This attribute specifies the number of columns spanned by the current cell.
    • If you don't include this attribute the browser defaults the COLSPAN to one ("1").
    • The value zero ("0") means that the cell spans all columns from the current column to the last column of the table.
  6. The rowspan attribute -- HTML 3.2 Standards     HTML 4.0 Standards     XHTML Standards
    • This attribute specifies the number of rows spanned by the current cell.
    • If you don't include this attribute the browser defaults the ROWSPAN to one ("1").
    • The value zero ("0") means that the cell spans all columns from the current column to the last row of the table.
