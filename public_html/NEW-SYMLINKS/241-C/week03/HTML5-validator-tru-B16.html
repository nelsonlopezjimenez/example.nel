Validating Your Code

One of the easiest ways to earn a great grade in these modules... validate your code! With HTML5, the standards are so much more relaxed than with XHTML 1.0, but you still need to make sure that your code works against the W3C's standards.

Lesson Objective: At the end of this lesson, you will be able to:

  • Explain the importance of consistently developing to a single W3C standard.
  • Create a Web page using the HTML5 standard.

 

So why validate?  My number one reason is that as you move forward through the topics of the certificate you will be working with parts and pieces of your Web documents through the Document Object Model (DOM).  Validation insures that you can access the different pieces of your document cleanly and easily. 

Number two, the best way to start having your pages render consistently across browsers is with standardized code.  Even IE is better when using W3C standards.

There are a lot more reasons, please read the article Why Validate? provided by the W3C.

 

Steps to validating your code at the W3C's validator by Direct Input:

  • Access the W3C's validator at http://localhost:22021/validator.w3.org
  • Select the Validate by Direct Input tab
  • Copy your full HTML document source code
  • Paste your HTML  document source code into the text area
  • Click the Check button
Read through the errors carefully, they will point to the line that has the error and describe the error that is the problem.  Warnings are of less concern, but you should be aware of them. 
Steps to validating your code at the W3C's validator by File Upload
  • Access the W3C's validator at http://localhost:22021/validator.w3.org
  • Select the Validate by File Upload tab
  • Click the Browse button and navigate to the location on your computer where you saved your file
  • Click the Check button
Read through the errors carefully, they will point to the line that has the error and describe the error that is the problem.  Warnings are of less concern, but you should be aware of them. 

 

If any of the error message seem like spaghetti soup to you, that's not unusual, check out http://localhost:22021/validator.w3.org/docs/help.html and of that doesn't help post the error message in the Learning Community along with the code causing the problem.
